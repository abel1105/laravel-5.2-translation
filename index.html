透過 Homestead 安裝 Laravel

什麼是 Homestead ?

現在，很多開發者使用虛擬機（VM）來開發動態網站跟應用程式。你可以在虛擬機上使用伺服器、資料庫與你的程式碼。你可以創建多個虛擬機實例在不同的專案下，如果你不需要這個虛擬機的時候，你可以放心的將它移除而不影響任何東西，你甚至可以重新創建一個虛擬機在幾分鐘內！

我們稱之為：「虛擬化」。

有非常多方法來達到虛擬化，但最流行的是 Oracle 的 VirtualBox 。 只要我們喜歡，VirtualBox 可以幫助我們安裝和運行多台虛擬機在Windows，Mac，Linux或Solaris的操作系統。然後，我們將使用一個叫做 Vagrant 的工具來管理與配置我們的虛擬開發環境。

在2014年，Laravel 的發明者，Taylor Otwell 引進了 Homestead 的方法。

Homestead 是一個以 Vagrant 為基礎的 Ubuntu 虛擬機 (VM)。它包含了所有我們需要用來開發 Laravel 應用程式的東西。意思是，只要我們安裝 Homestead，我們就已經有了 PHP、Nginx、資料庫與其他程式的一台虛擬機。我們就可以立即開始創建我們的 Laravel 應用程式。

下面是所有安裝的程式：

  • Ubuntu 14.04
  • PHP 5.6
  • HHVM
  • Nginx
  • MySQL
  • Postgres
  • Node (包含 Bower, Grunt, and Gulp)
  • Redis
  • Memcached
  • Beanstalkd
  • Laravel Envoy
  • Fabric + HipChat 擴充套件

Composer 是什麼？ Composer 是一個跨平台的 PHP 依賴性管理工具。 我們可以透過 Composer 來安裝、移除、更新 PHP 函式庫。你會透過本書來學到更多。現在，我們需要安裝 Composer 來使用它安裝 Homestead。

如果要安裝 Composer，我們必須使用 終端機（Terminal）應用程式，或是其他終端模擬器，例如：iTerms。在 Mac 上，你可以在應用程式->工具程式->終端機 找到它。

大部分你需要在 終端機(Terminal) 做的事，就是輸入一段指令後，然後按下 Return 鍵去執行指令。

讓我們開始透過執行命令來安裝 Composer：

curl -s https://getcomposer.org/installer | php

如果你看到下面這個錯誤：

The detect_unicode setting must be disabled.

改用下面這行命令

curl -s getcomposer.org/installer | php -d detect_unicode=Off

程式安裝過程中會在你的工作目錄下載 composer.phar。如果想要在系統中任何地方都能輕鬆存取 composer，你需要將 composer.phar 移動到 /usr/local/bin 這個資料夾，透過執行下面這個命令來達到：

sudo mv composer.phar /usr/local/bin/composer

如果他要求你的密碼，輸入你 Mac 使用者的密碼。

備註：在 OSX Yosemite，沒有預設的 /usr 資料夾。如果你看到下面這個錯誤：

/usr/local/bin/composer: No such file or directory

因此透過輸入下面的指令，來手動創建 /usr/local/bin 資料夾

sudo mkdir /usr/local/bin

vi ~/.homestead/Homestead.yaml

設定 Homestead

Homestead.yaml 這隻檔案的架構非常簡單，總共有七個部分，接下來一一介紹。

第一部分 - 設定 VM

ip: "192.168.10.10"
memory: 2048
cpus: 1
provider: virtualbox

你可以看到，我們可以設定 IP 位置、記憶體、CPUs 與 VM服務提供者。這部分不是那麼重要，所以可以保留預設的情況。

第二與第三部分 - 設定 SSH

authorize: ~/.ssh/id_rsa.pub

keys:
    - ~/.ssh/id_rsa

我們需要產生 SSH 金鑰來給 Homestead 授權使用者與連結 VM，如果你有用過 Git，你可能已經有 SSH 金鑰了。如果你沒有，簡單的跑下面這段程式碼來產生：

ssh-keygen -t rsa -C "you@homestead"

這個指令會自動產生你的 SSH 金鑰，並放到 ~/.ssh 資料夾中，你不需要再做任何事。

第四部分 - 設定共享資料夾

使用 folders 來具體說明我們要共享哪個資料夾給我們的 Homestead 環境。只要我們新增、編輯、更改任何文件在我們的本機端，在 Homestead VM 的檔案即會自動更新。

folders:
    - map: ~/Code
      to: /home/vagrant/Code

預設下，你可以看到目前是設定成 ~/Code 資料夾，而那個位置則是用來放本機端所有的檔案、腳本。如果你想要把你的程式碼放在別的地方，你可以自行修改成你想要的位置。

而 /home/vagrant/Code 則是在 VM 裡面放程式碼的資料夾。通常，我們不需要更動它。

第五部分 - 設置網域（domain）

sites:
    - map: homestead.app
      to: /home/vagrant/Code/Laravel/public

這個部分可以讓我們將網域指到我們在VM裡面的資料夾，舉例來說，我們可以將 homestead.app 指到在我們的 Laravel 專案裡的 public 資料夾，然後我們就可以簡單的進入我們的 Laravel 應用程式，透過這個網址："http://homestead.app"。

記住，每當你要新增任何網域，你必須在本機端上修改 hosts 檔案，才可以使請求打到我們的 Homestead 環境。

在 Linux 或 Mac的系統上，你可以在 etc/hosts 或 /private/etc/hosts 資料夾找到 hosts 檔案，你可以輸入下面的指令來修改 host 檔案：

sudo open /etc/hosts

如果你知道怎麼使用 VI 或 VIM，輸入下面的指令來修改它：

sudo vim /etc/hosts

在 Windows 裡，你可以在 C:\Windows\System32\drivers\etc\hosts 找到 hosts 檔案。

打開檔案後，你需要將下面這行文字，加在所有文字的最後面。

192.168.10.10  homestead.app

成功！ 當我們運行 Homestead 後，就可以透過這個網址進入網站。

http://homestead.app

請記住你可以依你所需更改網址（homestead.app）。

預設下，藉由 HTTP 通過 port 8000 與 HTTPS 通過 port 4300，即可以進入所有網站。

第六部分 - 設定資料庫

databases:
    - homestead

這是在 VM 裡面的資料庫名稱。通常，我們會保留預設。

第七部分 - 新增自定義變數

variables:
    - key: APP_ENV
      value: local

如果我們想要新增一些自定義變數在我們的 VM 上，我們可以新增在這裡。但這不太重要，我們移往比較有趣的部分。

安裝 Laravel 透過 Laravel Installer(安裝包)

這是推薦的方法。是比較新比較快的方法，你可以使用這個方法來創建 Laravel 應用程式。

首先，我們需要使用 Composer 來下載 Laravel installer。

composer global require "laravel/installer=~1.1"

這個指令是用來下載 Laravel 5.0，如果你要使用最新的版本，輸入下面的：

composer global require "laravel/installer"

注意： 推薦使用 Laravel 5.0 來學習基礎的 Laravel 框架，你之後可以更新到更高的版本。但是你也可以使用最新的版本，本書會經常地更新來支持新版本。

一旦下載完，你可以輸入下面的指令來創建一個新 Laravel 專案：

laravel new nameOfYourSite

你可以把 nameOfYourSite 改成任何你想要的，但記得要修改在 Homestead.yaml 的 sites section 以吻合你網站的名字，

瞭解 routes.php

其中一個在 Laravel 中最重要的檔案就是 routes.php。這個檔案可以在 Laravel/app/Http/ 這裡找到。

我們透過這檔案來指到路由。

這是什麼意思？

路由意思就是告訴 Laravel 去抓 URL 的請求，然後分配個別的請求，使其對應到你的希望的行為。舉例來說，當有人訪問 homestead.app ，也就是我們的首頁時，Laravel 會“想“：「歐，這個傢伙正要去我們的首頁，我需要展示歡迎頁面，並顯示出一些引文讓他閱讀！」

我們通常設定所有的路由在 routes.php 檔案裡。

改變 Laravel 首頁

如果要改變 Laravel 首頁，你需要修改 routes.php 這隻檔案。

讓我們來看有什麼在裡面。

Route::get('/', function () {
    return view('welcome');
});

我們只會有一個預設的路由。

這個路由告訴 Laravel 當有人使用了 GET 請求 到我們的 根網址（相當於 /）就會返回 歡迎視圖。

注意： 從 Laravel 5.1 開始，routes.php 已經變了。如果你看到不一樣的 routes.php 檔案，你可能會是舊的 Laravel 版本，請更新你的應用程式到版本 5.1 或更新。

routes.php （5.0版本） 

Route::get('/', 'WelcomeController@index');
Route::get('home', 'HomeController@index');
Route::controllers([
'auth' => 'Auth\AuthController',
'password' => 'Auth\PasswordController',
]);

我們剛剛使用了 匿名函數（anonymous function） 來修改我們的首頁。在 PHP 中，我們稱它為："閉包（Closure）"。

閉包是一個沒有名字的函數。

建議你應該每次都使用控制器，因為控制器幫你簡單結構化你的程式。舉例來說，你可以統整你全部的使用者行為在 UserController （使用者控制器）裡面，所有文章的行為就放在 PostsController （文章控制器）裡面。

為了更瞭解什麼是 控制器，我們會用控制器來創造一些頁面。

新建我們的第一個控制器

這裡有兩個方法來新增一個控制器：

手動的新增一個控制器

使用你最喜歡的程式編輯器（例如：PHPStorm、Sublime Text），創建一個新的檔案叫做 PagesController.php。

把創建的檔案放在 app/Http/Contorllers/ 資料夾。

把 PagesController.php 檔案更新成下面這樣：

<?php namespace App\Http\Controllers;

class PagesController extends Controller {

    public function home()
    {
        return view('welcome');
    }

}

太棒了！你現在有了一個裡面有首頁行為的頁面控制器。

你可以看到首頁行為告訴 Laravel "回傳歡迎視圖"：

return view('welcome');

使用 Artisan 創建控制器

你也可以使用 Artisan 來自動的產生控制器，而不是使用手動的新建控制器。

Artisan 是一個 Laravel 命令列工具，它協助我們做一些我們不喜歡手動做的事情。使用 Artisan，我們可以創造模型、視圖、控制器、資料庫遷移與其他的事情。

你已經知道怎麼使用 Terminal 或 Git Bash，輸入下面的指令來創建一個控制器：

php artisan make:controller PagesController

注意： 使用 vagrant ssh 進到你的虛擬機器, 再 cd 到你的 Laravel 資歷夾，然後再使用這個指令.

你會看到：

預設情況，Laravel 會創建一個 RESTful 控制器。因此，我們的 PagesController 類別會有下面幾個行為：

  1. index
  2. show
  3. create
  4. store
  5. edit
  6. update
  7. destroy

我們不需要這些，你可以直接移除他們，然後以下面的首頁行為範例來取代掉：

<?php namespace App\Http\Controllers;

use App\Http\Requests;
use App\Http\Controllers\Controller;

use Illuminate\Http\Request;

class PagesController extends Controller {

    public function home()
    {
        return view('welcome');
    }

}

或是，你可使用下面這個指令來產出一個沒有任何預設行為的 PageController ：

php artisan make:controller PagesController --plain

注意： 從 Laravel 5.2 開始，這個 "--plain" 的功能已經被移除。控制器會一直都是空白的。

做得好！你現在有個裡面有首頁行為的頁面控制器。

你可以看到首頁行為告訴 Laravel 去『回傳歡迎視圖』：

return view('welcome');

做得好！你剛剛用了自己的控制器來展現了首頁！

使用欲先編譯的 Bootstrap 檔案

Twitter Bootstrap 是用 Less 寫的，一個CSS的預處理器。

http://lesscss.org

不幸的是，瀏覽器並不認識 Less。你必須使用 Less 編譯器 來 編譯 所有 Less 檔案產生 CSS 檔案。當然，前提是你必須學會 Less。

幸運的是，Bootstrap 有提供編譯過的 CSS、JS 和字體給我們。我可以下載並使用他們，避免處理 Less 的檔案：

http://getbootstrap.com/getting-started

點擊 Download Bootstrap 來下載最新的編譯過的 Bootstrap 檔案。

解壓縮下載的 .zip 檔案，會看到三個資料夾：

  1. css
  2. js
  3. fonts

把他們全部放在你應用程式的 public 資料夾。（~/Code/Laravel/public）。

注意：預設情況下，Laravel 有建立 css 和 fonts 資料夾給我們。在 fonts 資料夾還包含所有 glyphicons 的字體。

http://gulpjs.com

Gulp 是一個以 Node.js 為底的基礎任務執行器，代表如果你要使用 Elixir，你必須要先安裝 Gulp 和 Node.js。

幸運的， Homestead 有把 Gulp 跟 Node.js 都預設裝好，我們可以立即使用 Gulp。如果你不想要使用 Homestead，你需要安裝 Node.js 和 Gulp。如果要安裝 Node.js，請到：

https://nodejs.org

npm install -g gulp

[14:09:33] CLI version 3.8.11

gulp

學習 Blade 模板

Balde 是一個官方 Laravel 的模板引擎。他非常厲害，而且只有簡單的語法。我們使用 Blade 來建立我們的 Laravel 應用程式排版。

@if ($product == 1)
 {!! $product->name !!}
@else 
There is no product!
@endif

等同於下面這樣的 PHP 程式：

if ($product ==1) {
echo $product->name;
} else {
echo("There is no product!");
}
